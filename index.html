What is a prime?¶

A prime number (or a prime) is a natural number greater than 1 that has no positive divisors other than 1 and itself.

— Wikipedia
In [2]:
def is_prime(n):
    # Greater than 1
    if n <= 1:
        return False
    # Any positive divisors (> 1 and < n)?
    for i in range(2, n):
        if n % i == 0:
            return False
    # If not, it must be a prime
    return True

Let's make it a bit more flexible¶

Currently we can only check if a particular number is prime.

Let's turn it into a generator function returning a sequence of primes.

This will allow us to:

  • Generate the first $n$ primes (take $n$ items from sequence)
  • Generate primes up to a certain size
  • Generate primes until a certain condition is met
  • Optimise it better later (we might not need to check every number)
In [5]:
from itertools import count

def first_try():
    # loop over all positive integers
    for i in count():
        if is_prime(i):
            # yield any that are primes
            yield i

Can we make it faster?¶

What about skipping all even numbers (apart from 2)?¶

In [9]:
from itertools import count

def skip_even():
    def is_prime(n):
        # check odd numbers >= 3 and < n
        for i in range(3, n, 2):
            if n % i == 0:
                return False
        return True
    yield 2
    # loop over odd integers from 3 upwards
    for i in count(3, 2):
        if is_prime(i):
            yield i

Can we reduce the number of checks further?¶

We could also skip multiples of 3¶

Would give us another factor of $\approx 3/2$ (not 3 because half of the numbers (even) are already skipped), but would also complicate the code. Can we do better?

Yes!¶

Factors always come in pairs: if $n$ has a factor $f$, then $g = n / f$ must also be a factor.
If $f \leq \sqrt n$ then $g \geq \sqrt n$, and vice versa (equal when $f = g = \sqrt n$).
So if $n$ has any prime factors, at least one of them must always be $\leq \sqrt n$,
and therefore we can stop checking once we reach $\sqrt n$.

In [11]:
from itertools import count
from math import sqrt

def to_sqrt():
    def is_prime(n):
        # check odd numbers from 3 to sqrt(n)
        for i in range(3, int(sqrt(n)) + 1, 2):
            if n % i == 0:
                return False
        return True
    yield 2
    for i in count(3, 2):
        if is_prime(i):
            yield i

Is this the best we can do?¶

We are still checking more numbers than necessary:
Once we've tested for divisibility by 3 and 5,
we shouldn't need to test their multiples (e.g. 9, 15, 25, 30, 45... etc).

i.e. we only need to check for divisibility by primes.

What about storing a list of primes so far, and only test dividing by those?¶

In [13]:
from itertools import count

def check_primes():
    yield 2
    primelist = []
    for candidate in count(3, 2):
        prime = True
        for p in primelist:
            if p * p > candidate:
                break # stop checking at sqrt(candidate)
            if candidate % p == 0:
                prime = False
                break
        if prime:
            yield candidate
            primelist.append(candidate)

What next?¶

Test dividing is (relatively) slow. Instead of test dividing candidate primes, we can generate and eliminate the composite numbers, leaving behind the primes.

The sieve of Eratosthenes¶

  1. start with a grid of numbers, from 2 to max_prime
  2. find first (next) unmarked number, return that as a prime
  3. mark all multiples of it (actually just from n onwards)
  4. go back to 2.
In [15]:
def sieve(max_prime):
    sieve = [True] * (max_prime // 2)
    yield 2
    for i in range(3, max_prime, 2):
        if sieve[i//2]:
            yield i
            for j in range(i*i, max_prime, i*2):
                sieve[j//2] = False

Problems?¶

Memory use¶

  • Use packed data structure (e.g. struct module), encode 8 cells/byte
  • Also skip multiples of 3 (only check numbers of form $6n \pm 1$)

Need to allocate storage upfront¶

Often don't know in advance how much to allocate (e.g. first 100k primes)

What can we do about it?¶

What about switching things around for each prime, we store the next multiple higher than the current candidate, then we just have to check if candidate is in the list, not multiple test divisions per candidate. For each multiple in the list, we store the original prime, so that when we reach it, we we can add it to generate the next multiple. But it could be a multiple of more than one prime, so we have to store a list of source primes:

In [17]:
from itertools import count

def unbounded_sieve():
    state = {}
    for candidate in count(2):
        if candidate in state:
            for factor in state[candidate]:
                if candidate + factor in state:
                    state[candidate + factor].append(factor)
                else:
                    state[candidate + factor] = [factor]
            del state[candidate]
        else:
            yield candidate
            state[2 * candidate] = [candidate]

from collections import defaultdict
from itertools import count

def improved_sieve():
    yield 2
    state = defaultdict(list)
    for candidate in count(3, 2):
        if candidate in state:
            for inc in state[candidate]:
                state[candidate + inc].append(inc)
            del state[candidate]
        else:
            yield candidate
            state[candidate * candidate] = [2 * candidate]
